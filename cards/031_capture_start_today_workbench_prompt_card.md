# Card 031 — Capture Start-Today Workbench prompt

## Status

Proposed / ready for Capture Back.

## Claim

The template Workbench should include a reusable prompt that generates the current best-practice "Start-Today Workbench" guide with copy and an infographic.

## Decision

Add a prompt and supporting LI that can generate an educational artifact explaining how Workbench operates today before a dedicated app, MCP endpoint, or native product surface exists.

Use "Start-Today Workbench" as the primary phrase.

"Stage 0" may be referenced only as a secondary internal/educational clarification.

## Why this matters

The Workbench is already usable today through:

- one chat window;
- two roles;
- local repo memory;
- terminal-based Capture Back;
- review-surfaced overlays;
- human commit;
- generated repo history for re-entry.

This should not be presented as a lesser demo. It is the first practical Workbench surface.

## Review standard

The generated guide should include:

- explanatory copy;
- a high-quality infographic;
- download-to-Downloads convention;
- terminal apply commands;
- paste-stdout-back-to-chat loop;
- human review gate;
- commit-after-review boundary;
- history-based re-entry.
