# Game 1 R16 Choice Menu Rule

Game 1 R16 winner picks are governed by resolved bracket adjacency, not by group eligibility.

An R16 slot may present a winner-pick menu only when both of its two source R32 slots have assigned teams. The menu choices must be exactly the two teams occupying those source R32 slots.

The R16 menu must not use the R32 eligible-group menu. It must not show all teams, all group teams, or third-place eligibility pools.

R16 pick state must be stored separately from R32 assignment state. R32 assignment answers preserve the Game 1 prediction layer; R16 answers represent the long-lived bracket-winner layer.

This rule is intentionally first-order. QF, SF, and Final Four advancement should follow the same resolved-source-slot pattern after R16 behavior is verified.
