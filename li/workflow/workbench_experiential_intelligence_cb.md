# Capture Back — Workbench Experiential Intelligence and Continuity Endpoints

## Insight

Workbench LI is not only continuity storage.

A Workbench grows by converting accepted work experience into **Experiential Intelligence**: future-facing guidance that can steer later Inference Infrastructure sessions.

In this framing:

```text
Experience
  -> Capture Back
  -> governed continuity
  -> distilled rules, cards, prompts, maps, invariants, workflows
  -> Experiential Intelligence
  -> guidance for future Inference Infrastructure
```

## Inference Infrastructure

Inference Infrastructure means the replaceable reasoning side of the work:

- LLMs
- agents
- orchestration
- tool routers
- MCP clients
- MCP servers
- future reasoning systems

The Workbench should not depend on stuffing all context into an II prompt. The prompt should be able to attach to a governed continuity endpoint.

```text
The prompt should not contain the Workbench.
The prompt should attach to the Workbench.
```

## Workbench endpoint pattern

A future Workbench MCP or IAM endpoint should expose the Workbench as governed, private continuity:

- MAP / read-first orientation
- authority order
- governing LI
- cards and capture-backs
- prompts and workflows
- repo history and accepted decisions
- privacy and custody rules
- draft-only write paths unless explicitly accepted by the human owner

The endpoint injects discipline into an II session by making governance the first returned context.

```text
The MAP is not just navigation.
The MAP is the boot contract.
```

## Ownership promise

The promise is not merely "own a pack" or "own a repository."

The promise is:

```text
Own the discipline.
Own the rules.
Own the privacy boundary.
Own the accepted experience that becomes guidance.
Let inference attach without letting it govern.
```

## Practical rule

Future Workbench changes should ask:

1. What experience did we just gain?
2. What should be preserved as raw continuity?
3. What should be distilled into future guidance?
4. What file becomes the boot surface for that guidance?
5. What should a future MCP/endpoint expose from this repo?
6. What remains private, draft-only, or human-approval-only?

## Short form

Workbench converts governed continuity into Experiential Intelligence.  
Experiential Intelligence guides future Inference Infrastructure.  
Inference can visit. It does not own the house.
