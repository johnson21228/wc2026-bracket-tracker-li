# Newcomer Infographic Contract

A Workbench LI repo SHOULD be able to generate a newcomer-facing infographic that explains the Workbench Loop without requiring the reader to understand implementation mechanics first.

## Required conceptual loop

```text
Pack → Reason → Capture Back → Verify → Commit + Repack → Repeat
```

## Required emphasis

The infographic SHOULD emphasize:

- human custody
- durable continuity
- git-backed memory
- LLMs as reasoning support
- small verified steps
- shareable packs

## Nomenclature rule

The visible loop step is **Capture Back**.

Do not use **Overlay** as the visible name of the loop step.

Overlay MAY be mentioned as one technical delivery mechanism for a Capture Back change.

Apply Command MAY be mentioned as the safe local execution path.

## Newcomer posture

The infographic is an introductory artifact, not the governing source.

If the graphic is wrong or unclear, fix the governing LI and prompt, then regenerate the graphic.
