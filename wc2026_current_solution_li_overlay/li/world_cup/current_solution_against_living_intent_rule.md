# Current Solution Against Living Intent Rule

## Purpose

This rule captures the product-level meaning of the World Cup Bracket Tracker Workbench.

The Workbench does not merely preserve continuity as an abstract asset. The continuity exists so the project can keep presenting the current best solution against the LI-stated goal.

## Principle

The Workbench output is:

```text
Current best solution
+
LI state that explains and governs it
```

The solution may be:

- a poster-derived data artifact
- a static HTML file
- a GitHub Pages site
- a hosted pool
- a scoring engine
- a reusable tournament platform

The LI states what the solution is trying to accomplish, what facts govern it, what is known, what is uncertain, and what should evolve next.

## World Cup proof case

This project demonstrates the mechanism:

```text
A photo of a poster
↓
structured schedule data
↓
static HTML tracker
↓
Game 1 Round-of-32 pick pool
↓
Game 2 knockout bracket pool
↓
results and scoring updates
↓
GitHub Pages or hosted site
```

Each stage is a real current solution. Each Capture Back updates the LI state so the next solution can remain aligned with reality.

## Product framing

Do not describe this Workbench as only an AI artifact generator.

Describe it as:

```text
Living intent → current solution.
```

or:

```text
A continuity system that keeps custom solutions current as facts, goals, and use change.
```

## Anti-slop rule

Generated artifacts are not accepted merely because they are generated.

A solution is accepted when it is aligned to:

- the stated goal
- the current source data
- the current rules
- the current user workflow
- the current release state

## Delivery rule

The deliverable is not only the file users interact with.

The deliverable is:

```text
the current solution
+
the Workbench state needed to continue evolving it
```

For this project, the public user may see only the HTML site, but the Workbench must preserve the data, LI, source history, scoring rules, and Capture Back needed to keep that site current.
