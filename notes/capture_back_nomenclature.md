# Capture Back Nomenclature

The Workbench Loop should name the conceptual act as **Capture Back**.

Capture Back means returning useful reasoning, decisions, context, evidence, and continuity from an LLM reasoning session back into the repo under human custody.

## Distinction

- **Capture Back** is the human / team / organizational continuity act.
- **Overlay** is one technical delivery mechanism for applying concrete repo changes.
- **Apply Command** is the safe local execution path.

The Workbench Loop is therefore:

```text
Pack → Reason → Capture Back → Verify → Commit + Repack → Repeat
```

Use **overlay** only when referring to the implementation mechanism.
Do not use **overlay** as the visible name of the Workbench Loop step.
