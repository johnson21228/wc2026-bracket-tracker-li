# Capture Back Review Surface Lesson

## Lesson

The user noticed that newer Capture Back overlays are more useful when the apply script and terminal workflow open the files that matter immediately after apply.

This is important because Capture Back is not just a file-transfer mechanism. It is a governed review workflow.

## Captured insight

A good overlay should make the human review path obvious:

- show what was added;
- explain why it matters;
- open the key docs, LI rules, and generated assets;
- verify and pack;
- leave commit as a deliberate human approval step.

## Product implication

Workbench should treat review-surfacing as part of the Capture Back protocol.

The overlay should not only update the repo; it should help the user inspect, accept, reject, or revise the update.

## One-line formulation

Apply below. Review above. Commit only after human approval.
