# Two-Pack Template Comparison Prompt Lesson

## Lesson

As Workbench evolves, target Workbenches may lag behind the generalized template. At the same time, target Workbenches may discover patterns worth harvesting back into the template.

The template needs a prompt for comparing two uploaded packs:

1. the current generalized template WB pack; and
2. a target WB pack.

## Captured insight

The comparison should be bidirectional:

- Template → Target: sync current generalized best practices.
- Target → Template: harvest reusable patterns after generalization.

## Why history matters

The latest `outputs/history/repo_history_for_llm_*.md` file in each pack should be read first when present.

The history file explains Workbench meaning: cards, Capture Backs, LI changes, docs, prompts, generated assets, and open state.

Raw file lists are not enough.

## Product implication

The template can become the update authority for future WBs, while still learning from successful target WBs.

## One-line formulation

The template teaches the target, and the target may teach the template — but only through explicit comparison, review, and generalization.
