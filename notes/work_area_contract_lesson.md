# Work Area Contract Lesson

During alignment of data-heavy Workbenches, default pack commands risked including generated run folders, sqlite databases, bulk jsonl outputs, logs, nested zip files, and other non-governance data.

The lesson: a Workbench repo may need a large working area, but the ordinary Workbench session pack should remain small, intentional, and II-consumable.

The pack is for governance and selected evidence. Bulk work becomes Workbench memory only after it is summarized, selected, and captured back.
