# Generate Initial Repo Structure

Given the intended Workbench LI purpose, propose a minimal repo structure.

Keep it small and usable.

Include folders only when they have a clear purpose.

Prefer:

- `source/` for source context
- `prompts/` for reusable LLM prompts
- `cards/` for continuity/work cards
- `docs/` for human-facing explanation
- `artifacts/` for generated durable outputs
- `validation/` for evidence, checks, review, or tests
- `outputs/history/` for generated repo-history artifacts
- `tools/` for local scripts

Explain what belongs in each folder.
