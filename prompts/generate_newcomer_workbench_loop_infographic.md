# Generate a Newcomer Workbench Loop Infographic

You are creating a clear, polished infographic for someone opening this Workbench LI repo for the first time.

Use the uploaded repo pack as source context. Do not invent repo concepts that are not supported by the pack.

## Goal

Create a visual explanation of the basic Workbench Loop for a newcomer.

The infographic should be suitable for a repo README, first-use document, website section, or introductory handoff.

It should look and feel like a clean modern technical diagram: wide horizontal flow, five numbered cards, simple icons, short explanatory text, arrows between steps, and a bottom strip that states the core promise.

## Required title

THE WORKBENCH (WB) LOOP

## Subtitle

A simple, repeatable cycle for building with LLMs while preserving durable continuity in a git-backed repository.

## Main loop steps

Use these five steps exactly as the conceptual loop:

1. PACK
   Start with the current repo pack.
   A clean, portable snapshot of the workbench and its governing LI.

2. REASON
   Reason with the model.
   Explore, plan, test language, compare options, and make decisions with LLM support.

3. CAPTURE BACK
   Preserve what matters.
   Move useful reasoning, decisions, context, notes, prompts, validation, or changes back into the repo.

4. VERIFY
   Check locally.
   Run the repo's verification commands and confirm the workbench remains coherent.

5. COMMIT + REPACK
   Make the repo the new starting point.
   Commit the verified change and generate the next pack.

## Important nomenclature

Do not use **Overlay** as the main loop step.

Use this distinction:

```text
Capture Back = the continuity act
Overlay = one possible technical delivery mechanism
Apply Command = the safe local execution path
```

The infographic may mention overlay only in small supporting text, such as:

```text
An overlay may be used to deliver a Capture Back change.
```

## Bottom message

Use this sentence or a close variant:

```text
ONE LOOP. DURABLE CONTINUITY. HUMAN CUSTODY.
```

Then add:

```text
Use the context you need. Capture back what matters. Keep building.
```

## Core principles strip

Include a small row labeled:

```text
GUIDED BY CORE PRINCIPLES
```

Use these items:

- Human custody — You decide. You capture. You own the record.
- LLM as support — Models help reason; they do not replace judgment.
- Git is memory — The repo is the durable source of truth.
- Small verified steps — Many small commits beat one big leap.
- Shareable by design — Clean packs make the workbench portable.

## Visual direction

Use a wide horizontal layout.

Style:

- clean modern technical diagram
- white background
- dark navy headline
- five cards across the page
- each card has a number, title, icon, and 2-3 short lines
- arrows between cards
- repeat arrow looping from Commit + Repack back to Pack
- bottom banner in dark navy
- professional but approachable
- suitable for a repo README or first-use document
