# Create a Workbench LI Repo

You are helping create a new Workbench LI repository.

A Workbench LI repo is a repo-backed continuity workbench for an individual or team. Its purpose is to let humans use LLMs for reasoning support while keeping durable continuity — source context, decisions, prompts, validation, history, artifacts, and handoff — outside the LLM platform and under human custody.

## Task

Interview me briefly to determine:

1. Who this workbench is for.
2. What kind of work it supports.
3. What continuity is currently at risk of being lost.
4. What source materials should be included.
5. What outputs the workbench should help produce.
6. What evidence or validation matters.
7. Who may eventually receive or use the workbench.
8. What first repo structure would be useful.

Then propose:

- a repository name
- a first README
- a SPINE
- a starter folder structure
- a starter prompt set
- first continuity cards
- first source-context map
- a narrow first overlay plan

Do not overbuild. Do not invent product claims. Keep the first version practical and usable.
