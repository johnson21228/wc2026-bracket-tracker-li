# Generate a Newcomer Multi-Pack Workbench Loop Infographic

You are creating a clear, polished infographic for a newcomer who needs to understand how the Workbench Loop works when more than one repo pack is used as context.

Use the uploaded repo pack as source context. Do not invent repo concepts that are not supported by the pack.

## Required title

THE WORKBENCH LOOP WITH MULTIPLE PACKS

## Subtitle

Same loop. Start with one or more packs. The loop does not change — your starting context can.

## Main loop steps

1. PACK(S)
   Start with one or more packs.
   Bring in the context you need.

   Examples:
   - Strategy WB
   - Client WB
   - Research WB

2. REASON
   Reason across the packs together.
   Explore, plan, and design with the full context.

3. CAPTURE BACK
   Capture the result into the target Workbench.
   Preserve useful reasoning, decisions, context, notes, validation, or changes in the repo that should own the continuity.

4. VERIFY
   Apply and verify locally.
   Check, test, and confirm the changes are correct.

5. COMMIT + REPACK
   Commit the target Workbench and repack.
   The target Workbench becomes the new starting point.

## Important nomenclature

Do not use **Overlay** as the main loop step.

Use this distinction:

```text
Capture Back = the continuity act
Overlay = one possible technical delivery mechanism
Apply Command = the safe local execution path
```

The infographic may mention overlay only in small supporting text, such as:

```text
An overlay may be used to deliver a Capture Back change.
```

## Bottom message

Use this sentence or a close variant:

```text
ONE LOOP. MULTIPLE PACKS. DURABLE CONTINUITY.
```

Then add:

```text
Use the context you need. Update the right Workbench. Keep building.
```
